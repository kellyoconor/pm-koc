## Customer Journey Map

Map the end-to-end customer experience from awareness through advocacy, identifying actions, emotions, pain points, KPIs, and improvement opportunities at each stage. This is a strategic artifact that combines customer empathy with business metrics to drive actionable improvements.

### Context

You are creating a customer journey map for **$ARGUMENTS**.

If the user provides files (interview transcripts, survey data, analytics, support tickets, or existing journey maps), read them first. Use web search to understand the product if a URL is provided.

### Key Concepts

**What this IS:** A strategic alignment and discovery tool that uncovers where the experience breaks down and where improvements have the greatest impact.

**What this is NOT:**
- **Not a user flow diagram** -- journey maps cover all touchpoints, not just product flows
- **Not a service blueprint** -- less detail on internal processes, more on customer experience
- **Not a user story map** -- story maps facilitate implementation; journey maps support discovery
- **Not static** -- journey maps evolve as customer behavior changes

### Prerequisites

Before mapping, ensure you have:
1. **A defined persona** -- you need to know who you're mapping (one map per persona)
2. **Buyer persona or proto-persona** with demographics, goals, challenges
3. **Touchpoint inventory** -- all places customers interact with your brand
4. **Research inputs** (ideal): interview notes, support tickets, analytics, churn data

If missing context, note gaps and plan research to fill them.

### Instructions

1. **Define the persona and scenario**
   Who is traveling this journey? Use a specific persona with JTBD, not a generic user. Define the scenario and goal (e.g., "New user onboarding from discovery to activation").

2. **Map the journey stages** (adapt to the product):

   | Stage | Description |
   |---|---|
   | **Awareness** | How do they first learn about the product? |
   | **Consideration** | What do they evaluate? What alternatives do they compare? |
   | **Decision/Acquisition** | How do they sign up or purchase? |
   | **Onboarding/Service** | First experience with the product -- time to value |
   | **Engagement** | Regular usage -- building habits |
   | **Retention** | What keeps them coming back? What might cause churn? |
   | **Advocacy/Loyalty** | When and why do they recommend the product to others? |

3. **For each stage, document**:

   - **Customer Actions**: What they do at this stage (be specific and observable)
   - **Touchpoints**: Where the user interacts with the product, brand, or team (website, email, in-app, support, social media, conferences, word-of-mouth)
   - **Thoughts & Questions**: What's on their mind ("Is this worth my time?" "How do I...?") -- use real customer quotes when available
   - **Emotions**: How they feel (excited, confused, frustrated, delighted). Be specific: not "happy" but "Relieved that setup took 30 minutes, not 3 hours as feared"
   - **Pain Points**: Friction, confusion, drop-off risks
   - **KPIs**: Metrics to measure success at this stage (stage-appropriate)
   - **Business Goals**: What you're trying to achieve at this stage
   - **Teams Involved**: Who owns this stage (marketing, sales, product, support, CS)
   - **Opportunities**: How to improve the experience at this point

4. **Identify critical moments**:
   - **Aha moment**: When the user first experiences core value
   - **Moments of truth**: Decision points where they commit or abandon
   - **Churn triggers**: Where users most commonly drop off

5. **Create the journey map table**:

   | Stage | Awareness | Consideration | Decision | Service | Loyalty |
   |---|---|---|---|---|---|
   | Customer Actions | ... | ... | ... | ... | ... |
   | Touchpoints | ... | ... | ... | ... | ... |
   | Customer Experience | ... | ... | ... | ... | ... |
   | KPIs | ... | ... | ... | ... | ... |
   | Business Goals | ... | ... | ... | ... | ... |
   | Teams Involved | ... | ... | ... | ... | ... |

6. **Analyze and prioritize improvements**:

   For each identified opportunity:
   - **Phase**: Which journey phase
   - **Pain Point**: Description
   - **Evidence**: Data/research supporting it
   - **Proposed Solution**: How to address
   - **Impact**: HIGH/MEDIUM/LOW with rationale

   Prioritize by:
   - **Impact**: How much would fixing this improve the customer experience?
   - **Feasibility**: How easy is this to fix?
   - **Alignment**: Does this support business goals?

### Workshop Mode

For interactive facilitation, ask up to 5 adaptive questions:

1. **Identify Actor**: Which persona or user segment?
2. **Define Scenario + Goal**: What is the actor trying to accomplish?
3. **Identify Journey Phases**: Generate 4-6 phases, ask for confirmation
4. **Map Actions, Thoughts, Emotions per Phase**: Generate 3-5 per phase, ask for refinement
5. **Identify Opportunities**: Generate 5-7 prioritized opportunities, ask for focus

### Common Pitfalls

- **Generic emotions**: "Customer feels happy" gives no insight. Be specific about why.
- **Missing touchpoints**: Include physical, digital, human, and automated -- not just your app.
- **Internal perspective**: Map what customers actually do, not what you want them to do. Validate with research.
- **No KPIs or goals**: Without metrics, no way to measure or prioritize.
- **One-and-done**: Review quarterly. Update based on new data and product changes.
- **Too many personas**: One map per persona. Mixing them creates confusion.
- **Created in isolation**: Facilitate with cross-functional team for shared mental model.

### Quality Checklist

Before finalizing, verify:
- [ ] One specific persona is identified
- [ ] All stages cover the full journey (awareness through advocacy)
- [ ] Both digital and physical touchpoints are included
- [ ] Emotions are specific and evidence-based, not generic
- [ ] Pain points are tied to evidence (analytics, research, tickets)
- [ ] KPIs and business goals are defined per stage
- [ ] Opportunities are prioritized by impact and feasibility
- [ ] Teams/owners are identified for each stage

Think step by step. Save as a markdown document. For visual journey maps, suggest the user create one in Miro or FigJam using this analysis as the foundation.
